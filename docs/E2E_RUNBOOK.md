# E2E Runbook

This document explains how to run the end-to-end test environment locally (docker-compose) and exercise the payment -> container creation flow using the HashPay-mock and smtp4dev.

Prerequisites
- Docker & docker-compose installed
- Ports 3000, 8081, 5000, 5025 available on the machine

Quick start
1. Copy .env.example to .env and set at minimum:
   - CLICD_BASE (default: http://51.159.67.180:8999)
   - CLICD_API_KEY (your CLICD API key; for tests you can use the demo key provided)

2. Start services:
   docker compose -f docker/docker-compose.e2e.yml up --build -d

3. Wait for services to start. The application will be available at http://localhost:3000. HashPay-mock UI: http://localhost:8081 (simple API). smtp4dev web UI: http://localhost:5000

4. Run the API test script to simulate a payment and check email delivery:
   bash tests/e2e/api_tests.sh

What the script does
- Calls HashPay-mock /simulate to post a webhook to your app (/api/payments/hashpay)
- The app receives the webhook and calls CLICD to create a container
- If user_email provided, the app attempts to send an email via SMTP (smtp4dev)
- The script then queries smtp4dev API to show the latest message

Artifacts
- Screenshots can be generated by adding Playwright tests (not yet included in this first patch). The API script prints JSON responses which you can save as files for verification.

Notes
- smtp4dev runs with no auth and exposes SMTP on port 5025 and web UI on 5000.
- The CLICD demo server at http://51.159.67.180:8999 is used by default. Keep your CLICD_API_KEY in .env or GitHub Secrets and never commit it.
